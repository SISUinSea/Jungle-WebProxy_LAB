# 소켓이 뭐냐?
OS 입장에서 소켓은 다른 컴퓨터 또는 컴퓨터 안의 다른 프로세스와 통신할 수 있도록 하는 끝점이다. ~~OS 입장에서는 프로그램에게 제공해야 하는 네트워크 인터페이스이다.~~
유닉스 프로그램의 입장에서 소켓은 그냥 여느 파일과 다르지 않다. (I/O가 가능한 파일 객체처럼 취급한다.)

# socket 함수
소켓을 만든다. 소켓을 만든 뒤 socket의 file descriptor를 반환한다.
서버와 클라이언트 모두 이 함수를 사용해서 소켓을 만든다. 이것 만으로 완전히 동작하는 소켓이 만들어지지 않는다.

# connect 함수
클라이언트가 서버와 연결을 위해서 수행하는 함수이다. 주소를 인자로 넘겨주어 해당 주소를 가진 서버와의 연결을 수행한다. 해당 작업을 완료하거나 에러가 날 때까지 프로세스를 멈춘다(Block). 이 함수가 성공적으로 완료되면 이제 서버로부터 데이터를 받거나 서버에 데이터를 보낼 수 있다.

# bind
서버를 위해서 필요한 함수이다. 
어떤 addr(IP 주소, Port Number)를 어떤 socket으로 받을지 연결한다.

# listen
기본적으로 소켓을 생성하면 active socket으로 생성된다. 해당 소켓을 정보를 어디론가 요청하지 않고, 대신 특정 주소로 들어온 요청을 처리하기 위한 수동적인 socket으로 만들기 위해서 수행하는 함수이다. int backlog를 인자로 넘겨주어 해당 서버가 몇 개의 요청을 큐에 쌓아 둘지에 대한 힌트를 준다.

# accept 함수
accept 함수가 실행되면 listenfd에 요청이 들어오는 것을 기다리고 있다가 listenfd에 요청이 생겼다면 해당 요청과 새로운 fd를 연결하고, 연결된 소켓 fd를 반환한다.
